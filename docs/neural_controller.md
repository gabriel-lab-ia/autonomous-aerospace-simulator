# Experimental Neural Controller

The neural control layer is an optional PyTorch component that implements the
simulator's existing `compute_throttle(state)` controller contract. Its output
is always clamped to `[0, 1]` before entering the simulation loop.

The current network is trained through supervised neural-controller
pretraining on synthetic telemetry generated by an explicit rule-based
teacher. This is useful for validating data, checkpoint, and inference
plumbing. It is not high-fidelity aerospace validation and does not establish
that the learned policy can land the simulated rocket reliably.

The training script can also consume preprocessed trajectories produced by the
controller benchmark. Preprocessing maps telemetry into the network's
13-dimensional sensor vector and retains controller throttle as an imitation
target:

```bash
uv run python scripts/benchmark_controllers.py
uv run python scripts/prepare_neural_training_data.py
uv run python scripts/train_neural_controller.py \
  --dataset outputs/neural_controller/preprocessed_controller_telemetry.csv
```

These are reproducible simulated data, not real aerospace flight data. Future
RL error penalties are designed in [the RL environment plan](rl_environment_plan.md)
but are not implemented or executed.

## Install And Train

```bash
uv sync --group ml
uv run python scripts/train_neural_controller.py
```

The training run writes an ignored checkpoint and metrics under
`outputs/neural_controller/`.

## Run In The Simulator Loop

```bash
uv run python scripts/run_neural_controller_simulation.py \
  --checkpoint outputs/neural_controller/neural_controller.pt
```

For integration testing only, random weights can be enabled explicitly:

```bash
uv run python scripts/run_neural_controller_simulation.py --allow-untrained
```

The script prints a warning in this mode because random inference is not a
meaningful control result. PyTorch remains outside the simulator's base
dependencies.
